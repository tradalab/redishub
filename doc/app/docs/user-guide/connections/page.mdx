# Connection Management

RedisHub provides a versatile connection manager designed to handle complex networking and security configurations across Windows, Linux, and macOS.

## Adding a New Connection

1. Click on the **+** (Plus) icon in the sidebar.
2. The connection dialog is organized into five specialized tabs:
    - **General**: Core configuration (Mode, Address, Credentials).
    - **Optional**: Performance tuning and NAT traversal.
    - **Proxy**: HTTP/SOCKS5 proxy settings.
    - **TLS/SSL**: Encryption and certificate management.
    - **SSH Tunnel**: Secure access to private networks.

![New Connection Dialog](/assets/en/new-connection.png)

## Connection Modes

### Standalone
The standard connection type for a single Redis instance.
- **Network**: Supports both **TCP** (standard) and **Unix Socket** connections.
- **Authentication**: Supports standard Redis password and ACL usernames.

### Cluster
Optimized for high-performance Redis Clusters.
- **Seed Address Import**: Use the "Import from Standalone" link to quickly copy your primary host/port as a seed address.
- **Automatic Discovery**: RedisHub automatically discovers the entire cluster topology from your seed nodes.
- **Slot Mapping**: Intelligent routing ensures commands are sent to the correct master nodes.

### Sentinel
Designed for high-availability setups.
- **Master Group Name**: Specify the name of the monitored master set (e.g., `mymaster`).
- **Sentinel Nodes**: List the addresses of your Sentinel instances for reliable discovery.
- **Dual Credentials**: Support for separate Sentinel-level and Master-level authentication.

## Advanced Networking

### SSH Tunnel
Connect to Redis servers located within private VPCs or behind firewalls.
- **Dynamic Routing**: RedisHub uses a dynamic SSH dialer that can reach any discovered node (Sentinel replicas, Cluster shards) through a single SSH tunnel.
- **Authentication**: Compatible with password, private key (PEM), and SSH agent.

![SSH Configuration](/assets/en/ssh-configuration.png)

### Proxy Support (HTTP/SOCKS5)
Route your Redis traffic through intermediate proxy servers.
- **Protocols**: Supports both **HTTP** and **SOCKS5**.
- **Authentication**: Supports proxy-level credentials (User/Pass).

![Proxy Configuration](/assets/en/proxy-configuration.png)

### TLS/SSL & mTLS
Ensure end-to-end encryption for your data.
- **Verification**: Validate server identity using CA certificates.
- **Mutual TLS (mTLS)**: Enhanced security requiring both server and client certificate validation.
- **SNI**: Support for Server Name Indication for virtual hosted targeting.

![TLS Configuration](/assets/en/tls-configuration.png)

## Tuning & Optimization (Optional Tab)

Access these settings under the **Optional** tab for fine-grained control:

### Performance Tuning
- **Execute Timeout**: Maximum time allowed for command execution before timing out.
- **Dial Timeout**: Timeout for establishing the initial network connection.
- **Key Load Size**: Limit the number of keys loaded during scan to ensure UI responsiveness in massive databases.

### Address Mapping (NAT Traversal)
Essential for environments where Redis nodes report internal, unreachable IPs (e.g., Docker or AWS ElastiCache behind NAT).
- **Mapping**: Define rules to translate internal `ip:port` to host-reachable `ip:port`.
- **Syntax**: `internal:port=external:port`, one entry per line.
