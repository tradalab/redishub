# Connection Management

RedisHub provides a versatile connection manager designed to handle complex networking and security configurations across various Redis architectures.

## Adding a New Connection

1. Click on the **+** (Plus) icon in the sidebar.
2. Select your **Connection Type** (Standalone, Cluster, or Sentinel).
3. Fill in the connection details:
    - **Name**: A descriptive name for the instance.
    - **Address**: The host and port (e.g., `127.0.0.1:6379`).

## Supported Architectures

### Standalone
The standard connection type for a single Redis instance. Ideal for development and simple production environments.

### Cluster
Optimized for high-performance Redis Clusters. 
- **Automatic Discovery**: Simply provide one or more seed nodes; RedisHub will automatically discover the entire cluster topology.
- **Slot Mapping**: Intelligent routing of commands to the correct master nodes.

### Sentinel
Designed for high-availability setups.
- **Master Name**: Specify the monitored master set name.
- **Sentinel Nodes**: List the addresses of your Sentinel instances for reliable master discovery and failover handling.

## Security & Advanced Networking

### SSH Tunneling
Securely connect to Redis servers located within private networks using standard SSH protocols.
- **Authentication**: Supports password, private key (with passphrase), and SSH agent.
- **Port Forwarding**: Robust internal management of local port mapping.

### TLS/SSL & mTLS
Ensure end-to-end encryption for your data.
- **One-way TLS**: Validate server identity using CA certificates.
- **Mutual TLS (mTLS)**: Enhanced security requiring both server and client certificate validation.

### Performance Tuning
Configure **Dial Timeout** and **Execute Timeout** in the *Optional* tab to handle high-latency or high-load environments.
