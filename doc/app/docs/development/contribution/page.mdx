# Contribution Guide

Thank you for your interest in contributing to RedisHub! We welcome contributions to help improve the project.

## Contribution Workflow

1. **Fork and Clone**: Fork the repository on GitHub and clone it locally.
2. **Branching**: Create a feature branch off of `main`.
3. **Coding Standards**:
    - **Go**: Follow `gofmt` and standard Go idioms.
    - **Frontend**: Use Tailwind CSS for styling and TypeScript for typed props.
4. **Testing**:
    - Ensure your changes run correctly in development mode using `bash scripts/dev.sh`.
5. **Commit Messages**: Use descriptive commit messages following the [Conventional Commits](https://www.conventionalcommits.org/) format: `<type>(<scope>): <description>`.
    - `feat:`: New feature.
    - `fix:`: Bug fix.
    - `docs:`: Documentation changes.
    - `refactor:`: Code changes that neither fix a bug nor add a feature.
    - `chore:`: Maintenance tasks (e.g., updating dependencies).

## Pull Request Process

1. Submit a Pull Request targeting the `main` branch.
2. Provide a clear description of the problem solved or the feature added.
3. Link to any relevant issues or discussions.
4. Once your code is reviewed and approved, it will be merged by a maintainer.

## Communication

If you have questions about the project or want to discuss a new feature, feel free to open a **GitHub Issue** or start a **Discussion**.
