# Commands Reference

RedisHub uses a `Makefile` to orchestrate common development, testing, and build tasks. All commands should be run from the project root.

---

## Development Cycle

| Command | Action |
|---------|--------|
| `make dev` | Run the app with the Next.js dev server + HMR (`scorix dev`) — main workflow |
| `make generate` | Regenerate handlers/types from proto + sqlx models from schema (`scorix generate proto/model`) |
| `make deps` | Install all backend and frontend dependencies |
| `make shell-install` | Install frontend dependencies only |
| `make doctor` | Check the toolchain & runtime deps (`scorix doctor`) |

---

## Testing & Quality

| Command | Action |
|---------|--------|
| `make test` | Run Go backend tests |
| `make lint` | Run complete linting (Go vet + ESLint) |
| `make lint-go` | Run Go vet only |

---

## Redis Infrastructure (Docker)

| Command | Action |
|---------|--------|
| `make redis-up` | Start all Redis containers (Standalone, Sentinel, Cluster) |
| `make redis-down` | Stop and remove Redis containers |
| `make redis-init-cluster` | Initialize cluster sharding (One-time only) |
| `make redis-status` | Show container status |
| `make redis-logs` | Tail logs of all Redis containers |

---

## Build & Release

| Command | Action |
|---------|--------|
| `make build` | Build the single production binary for the host (`scorix build`) |
| `make package` | Build + wrap into a native installer — msi/dmg/appimage per `scorix.yaml` (`scorix package`) |
| `make clean` | Remove build artifacts and dist |

---

> [!TIP]
> If a build fails, run `make doctor` to check the toolchain.
