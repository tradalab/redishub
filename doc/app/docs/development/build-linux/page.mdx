# Building for Linux

Linux artifacts (`.AppImage`) are built natively with the `scorix` CLI — no Docker required.
Packaging the AppImage needs `linuxdeploy` and the GTK/WebKit dev libraries.

## Prerequisites

- **Go 1.26+** and **Node.js 22+** + **pnpm**
- **AppImage tooling** — `linuxdeploy` plus `libgtk-3-dev`, `libwebkit2gtk-4.0-dev`, `libfuse2`, `patchelf`, `squashfs-tools`
- **scorix CLI** — `go install github.com/tradalab/scorix/cmd/scorix@latest`

## Build Process

From the project root:

```bash
make package          # scorix package — builds the binary + wraps it into an AppImage
```

Or invoke `scorix` directly to target an explicit OS/arch (this is what CI runs):

```bash
scorix package -d . --os linux --arch amd64
```

## Output Locations

- `artifacts/RedisHub-1.11.0-linux-amd64.AppImage` (version tracks `app.version` in `scorix.yaml`)

This AppImage runs directly on any modern Linux distribution without installation.

> [!NOTE]
> CI runs the same `scorix package` step on a native `ubuntu-22.04` runner — see
> `.github/workflows/release-linux.yml`.
