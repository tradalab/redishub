# Troubleshooting

Common development, setup, and build issues for RedisHub, with solutions.

---

## `//go:embed all:.scorix/dist` Fails at Build Time
**Symptom**: A bare `go build` / `go run` fails with an error about missing files in `.scorix/dist/`.

**Solution**: Build through the `scorix` CLI — `make dev` (or `make build`) compiles the Next.js frontend
into `.scorix/dist` before the Go build. For `make test` / `make lint`, the Makefile builds the shell
once automatically (see the `_ensure-embed` target), so you don't need to do it by hand.

---

## App Window Doesn't Open
**Symptom**: `make dev` completes without error, but no application window appears.

**Solution**:
1.  Check the logs in your terminal for any hidden errors.
2.  **Windows**: make sure the [WebView2 Evergreen Runtime](https://developer.microsoft.com/microsoft-edge/webview2/) is installed (preinstalled on Windows 11 / recent Windows 10).
3.  **Linux**: install the GTK/WebKit runtime — `sudo apt-get install -y libgtk-3-0 libwebkit2gtk-4.1-0`.
4.  Run `make doctor` (`scorix doctor`) to validate the toolchain and runtime dependencies.
5.  Enable debug mode in `scorix.yaml` to open DevTools automatically:
    ```yaml
    window:
      debug: true
    ```

---

## Redis Cluster Won't Connect
**Symptom**: You can connect to standalone Redis but not to the local Docker cluster.

**Solution**:
1.  Ensure you have run `make redis-up`.
2.  Run `make redis-init-cluster`. The Redis cluster requires a one-time initialization of its sharding slots before it can accept connections.

---

## Port Conflict
**Symptom**: `make dev` or `make redis-up` fails with "address already in use".

**Solution**:
1.  Identify the conflicting port (e.g., `8080` for the web server or `6379` for Redis).
2.  Edit the port mappings in `dev/docker-compose-*.yaml` or `scorix.yaml` (`web.port`).
3.  Update the connection settings in the RedisHub UI to match the new ports.
