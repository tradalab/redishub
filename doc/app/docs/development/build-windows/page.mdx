# Building for Windows

Windows artifacts (`.msi`) are built natively with the `scorix` CLI — no Docker required. The MSI
step needs the WiX Toolset.

## Prerequisites

- **Go 1.26+** and **Node.js 22+** + **pnpm**
- **WiX Toolset v6** — `dotnet tool install --global wix --version 6.0.2`
- **scorix CLI** — `go install github.com/tradalab/scorix/cmd/scorix@latest`

## Build Process

From the project root:

```bash
make package          # scorix package — builds the binary + wraps it into an MSI
```

Or invoke `scorix` directly to target an explicit OS/arch (this is what CI runs):

```bash
scorix package -d . --os windows --arch amd64
```

Code signing is applied automatically when the signing secrets from `scorix.yaml`
(`sign.windows`) are present in the environment; otherwise an unsigned MSI is produced.

## Output Locations

- `artifacts/RedisHub-1.11.0-windows-amd64.msi` (version tracks `app.version` in `scorix.yaml`)

> [!NOTE]
> CI runs the same `scorix package` step on a native `windows-2022` runner — see
> `.github/workflows/release-windows.yml`.
